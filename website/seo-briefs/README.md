## What's In Each Brief

Each `.md` file contains:

### 1. **Current Performance**

- Average position in Google
- Impressions count

### 2. **Keyword Strategy**
- Primary keyword (matched from high-impression queries)
- Supporting keywords (3-5 related terms)

### 3. **On-Page Optimization**
- **Title tag** (≤60 chars) - SEO-optimized, includes brand
- **H1 heading** (≤70 chars) - Aligned to primary keyword
- **Meta description** (150-160 chars) - Compelling, includes CTA

### 4. **Content Recommendations**
- Intro paragraph guidance (intent-focused)
- 3-5 suggested subheadings covering user sub-intents
- 3-6 FAQ pairs with answers

### 5. **Technical SEO**
- **HTTPS Normalization** - Checks for http:// URLs
- **Internal linking** - 2-4 strategic links to Services/Insights
- **JSON-LD FAQPage** - Ready-to-deploy structured data

## Google Best Practices Applied

✅ **Title Tags**
- Under 60 characters
- Primary keyword at start
- Brand at end
- No keyword stuffing

✅ **Meta Descriptions**
- 150-160 characters
- Actionable language
- Includes benefit/outcome
- Natural keyword placement

✅ **Headings**
- H1 matches primary intent
- Subheads answer user questions
- Hierarchical structure (H1 → H2 → H3)

✅ **Content**
- Intent-first approach
- Answer questions early
- Use clear, concise language
- Add FAQs for voice search

✅ **Technical**
- HTTPS everywhere (canonical, schema, links)
- Structured data (FAQPage)
- Internal linking with context
- Mobile-friendly formatting

✅ **E-E-A-T Signals**
- Author/publisher in schema
- Updated dates shown
- Citations to supporting content
- Clear expertise indicators

## Regeneration

To regenerate with fresh GSC data:

```bash
# 1. Export new data from Google Search Console
#    - Performance → Queries (download as Queries.csv)
#    - Performance → Pages (download as Pages.csv)

# 2. Place in gsc_data/
cp ~/Downloads/Queries.csv gsc_data/
cp ~/Downloads/Pages.csv gsc_data/

# 3. Run generator
python3 tools/gsc_to_briefs.py

# 4. Review new briefs
ls -lt website/seo-briefs/*.md | head -10
```

## Quality Checklist

Before deploying changes, verify:

- [ ] Title ≤60 chars, includes primary keyword
- [ ] Meta description 150-160 chars, compelling
- [ ] H1 matches search intent
- [ ] Content answers user questions in first 2 paragraphs
- [ ] FAQ section added with 3-6 Q&As
- [ ] FAQPage JSON-LD added to page
- [ ] Internal links use descriptive anchor text
- [ ] All URLs use https://
- [ ] Canonical tag points to https version
- [ ] Mobile-friendly formatting
- [ ] Page loads in <2.5 seconds (Core Web Vitals)
