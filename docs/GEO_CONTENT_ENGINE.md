# GEO Content Engine — Layer 7

**Status:** OPERATIONAL FRAMEWORK  
**Purpose:** Repeatable content generation from observable inputs

⸻

## Input Sources

### 1. GSC Anomalies
- Indexing failures
- Coverage issues
- Performance drops
- Query pattern changes

**Mapping:** Anomaly → Failure Mode

### 2. AI Overview Behavior
- Content appearing/disappearing
- Citation patterns
- Source selection
- Answer quality changes

**Mapping:** Observation → Field Note

### 3. Indexing Failures
- Pages not indexed
- Duplicate content issues
- Canonical problems
- Schema errors

**Mapping:** Failure → Failure Mode

### 4. Client Audits
- Retrieval gaps
- Citation failures
- Confidence issues
- Signal problems

**Mapping:** Pattern → Fundamental Update

### 5. Tool Telemetry
- Retrieval scores
- Citation eligibility
- Confidence metrics
- Segment quality

**Mapping:** Data → Field Note

⸻

## Output Mapping

### Anomaly → Failure Mode

**Process:**
1. Identify observable anomaly
2. Document what the model sees
3. Explain why confidence drops
4. List what gets ignored
5. Document common triggers
6. Record observed outcomes
7. Provide mitigation strategy

**Template:** Use failure mode template

**Example:** GSC shows duplicate content → Canonical Drift failure mode

⸻

### Observation → Field Note

**Process:**
1. Document what was observed
2. Note constraints and conditions
3. Record measurable outcomes
4. Avoid speculation
5. Use neutral tone

**Template:** "We observed X behavior across Y surfaces under Z constraints."

**Example:** AI Overviews cite different sources → Field note on citation patterns

⸻

### Pattern → Fundamental Update

**Process:**
1. Identify recurring pattern
2. Document mechanics
3. Update fundamental page
4. Link to related concepts
5. Maintain consistency

**Template:** Update existing fundamental page with new mechanics

**Example:** Consistent retrieval failures → Update "How Generative Engines Retrieve"

⸻

## Content Slotting

**Rule:** Each new insight slots into the IA, never expands it.

**Process:**
1. Identify appropriate IA location
2. Create content following template
3. Link to related concepts
4. Update index if needed
5. Never create new top-level categories

**Rationale:** Stable IA increases retrieval confidence. Expansion fragments authority.

⸻

## Automation Framework

### Step 1: Input Collection
- Monitor GSC, AI Overviews, indexing, audits, telemetry
- Document anomalies, observations, failures, patterns, data

### Step 2: Classification
- Map to failure mode, field note, or fundamental
- Identify appropriate IA location

### Step 3: Content Generation
- Follow template for content type
- Apply governance rules
- Implement required schema

### Step 4: Linking
- Link to related concepts
- Update indexes
- Maintain retrieval graph

### Step 5: Publication
- Review against governance checklist
- Verify schema implementation
- Publish and monitor

⸻

## Maintenance

**Weekly:**
- Review input sources
- Classify new observations
- Generate content as needed

**Monthly:**
- Audit content for governance compliance
- Update indexes
- Verify internal linking

**Quarterly:**
- Review IA structure
- Consolidate duplicate concepts
- Update fundamentals with new mechanics

⸻

**This framework ensures content generation remains mechanical and repeatable.**
